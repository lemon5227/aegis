# Aegis MVP Plan

## 目标
在最短时间内交付一个可演示、可联调、可迭代的去中心化论坛 MVP。

## Phase 1: 项目骨架（1-2 天）
- 初始化 Wails + React + Go 项目
- 建立身份生成与签名验签接口
- 前端展示本机公钥

## Phase 2: 本地数据层（1-2 天）
- 初始化 SQLite
- 建立消息表与治理状态表
- 实现 100MB 双分区配额控制（私有区/公共区）

## Phase 3: P2P 帖子流（2-3 天）
- 接入 libp2p + gossipsub
- 实现帖子广播与接收
- 前端时间线展示

## Phase 4: Shadow Ban 闭环（2 天）
- 管理端可发 SHADOW_BAN / UNBAN
- 收到治理消息后更新本地状态
- 被封禁账号实现“自己可见、他人不可见”

## Phase 5: 最小风控（1 天）
- 无效签名直接丢弃
- 消息大小上限
- 单 peer 基础限速

## MVP 验收
- 三个节点联调：普通用户A、普通用户B、管理节点
- A 与 B 可互相看帖
- 管理节点封禁 A 后：A 本机仍可见自己新帖，B 不可见
- 解封 A 后：A 新帖恢复被 B 看见

## 暂不纳入 MVP
- 投票治理执行器
- 存储义务可验证挑战机制
- 复杂经济激励与隐私增强网络
